## Directory Purposes

**`src/`:**
- Purpose: TypeScript source code for frontend
- Contains: Main entry point, terminal emulator logic, interactive features, utilities
- Key files: `index.ts` (scene init), `terminal/*` (shell/xterm/games)

**`src/terminal/`:**
- Purpose: Terminal emulation and interactive features
- Contains: Core shell logic, xterm integration, games (vim, piano), theme switcher, audio
- Key files: `ShellEmulator.ts` (command dispatch), `XTermAdapter.ts` (event handling)

**`api/`:**
- Purpose: Serverless backend endpoints (Vercel Functions)
- Contains: Claude AI chat handler, rate limiting, system prompt
- Key files: `chat.ts` (main endpoint), `system-prompt.ts` (AI personality)

**`assets/`:**
- Purpose: Static media—images, audio, documents
- Contains: Favicon, apple icon, keyboard sound samples, ambient music, resume PDF
- Key files: Audio in `audio/`, images in `images/`, content in `content/`

**`dist/`:**
- Purpose: Build output (not committed to git)
- Contains: Minified JavaScript bundle generated by tsup
- Key files: `index.global.js` (loaded by HTML)

**`.planning/`:**
- Purpose: GSD project documentation (generated, not committed)
- Contains: Codebase analysis documents (ARCHITECTURE.md, STRUCTURE.md, etc.)

## Key File Locations

**Entry Points:**
- `index.html`: Static HTML shell, loads `dist/index.global.js` script
- `src/index.ts`: Application bootstrap—three.js scene, rendering loop
- `api/chat.ts`: Backend API handler for AI chat requests

**Configuration:**
- `tsconfig.json`: TypeScript compiler options (ESNext target, bundler module resolution)
- `tsup.config.ts`: Build tool config (input: src/, output: dist/, minified)
- `vercel.json`: Deployment config (buildCommand, outputDirectory, CORS headers)

**Core Logic:**

**Interactive Features:**

- `src/terminal/piano.ts`: Piano synthesizer mini-game
- `src/terminal/theme.ts`: CRT effect theme selector (bloom, scanlines, etc.)
- `src/terminal/resume.ts`: Resume display with download option

**Utilities:**
- `src/utils.ts`: isMobileDevice() detection
- `src/terminal/audio.ts`: Audio context management, sound playback

**Assets:**
- `assets/audio/press_key[1-5].mp3`: Keyboard click sounds (5 variations)
- `assets/audio/delete.wav`: Backspace sound
- `assets/audio/background.mp3`: Ambient background loop
- `assets/images/favicon.png`: Browser tab icon
- `assets/content/resume.pdf`: Resume PDF file

**Backend Configuration:**
- `api/system-prompt.ts`: PROMETHEUS personality (280+ lines of detailed context)
  - Note: Gitignored in production; set as SYSTEM_PROMPT env var in Vercel

## Naming Conventions

**Files:**
- `[Feature].ts`: Feature modules (camelCase)
  - Examples: `XTermAdapter.ts`, `ShellEmulator.ts`, `theme.ts`, `piano.ts`
- PascalCase for class/interface-heavy modules (`XTermAdapter.ts`, `ShellEmulator.ts`)
- camelCase for utility/feature modules (`theme.ts`, `piano.ts`, `audio.ts`)

**Directories:**
- camelCase for organizational directories: `src/terminal/`, `assets/audio/`, `.planning/codebase/`

**Code Entities:**
- **Classes**: PascalCase (`XTermAdapter`, `Terminal`)
- **Functions**: camelCase (`isMobileDevice`, `playKeyClick`, `getPrompt`)
- **Interfaces**: PascalCase (`TerminalIO`, `CommandContext`, `Theme`, `KeyHandler`)
- **Types**: PascalCase (`CommandHandler`, `KeyHandler`)
- **Constants**: UPPER_SNAKE_CASE when global (`PASTE_COOLDOWN_MS`, `RATE_LIMIT_MAX`)
- **Exports**: Named exports preferred, index reexports (no barrel files yet)

## Where to Add New Code

**New Feature (Built-in Command):**
- Primary code: `src/terminal/ShellEmulator.ts`
  - Add handler: `registerCommand("newcmd", (ctx) => { ... })`
  - Handler receives TerminalIO for output, args for parameters
- If complex (game-like): Extract to new file `src/terminal/[feature].ts`
  - Example: `piano.ts` has `runPiano()` exported, called from ShellEmulator
- Tests: Not currently set up (npm run test is placeholder)

**New API Endpoint:**
- Implementation: Create `api/[name].ts` following Vercel Functions pattern
- Pattern: `export default async function handler(req, res)`
- Access rate limiting: Use Vercel KV (already configured in chat.ts)
- System prompt: If AI-related, load via getSystemPrompt() utility

**Utilities/Helpers:**
- Shared helpers: `src/utils.ts` (currently mobile detection)
- Terminal-specific helpers: `src/terminal/[module].ts`
- API-specific helpers: Extract to separate file in `api/` directory

**Styles:**
- Currently: Inline CSS in `index.html` (minimal—just container sizing)
- Audio/visual: Handled by three.js/cool-retro-term-renderer via JavaScript properties

**Assets:**
- Audio: Place `.mp3`/`.wav` files in `assets/audio/`
- Images: Place `.png`/`.jpg` in `assets/images/`
- Documents: Place in `assets/content/`
- Reference in code: Use absolute paths `/assets/...` (resolved from root)

## Special Directories

**`dist/`:**
- Purpose: Build output directory
- Generated: Yes (by `npm run build`)
- Committed: No (.gitignore)
- What to know: Contains minified `index.global.js` loaded by HTML

**`node_modules/`:**
- Purpose: Dependencies
- Generated: Yes (by `npm install`)
- Committed: No (.gitignore)
- Key packages: three.js, xterm, cool-retro-term-renderer, @anthropic-ai/sdk, @vercel/kv

**`api/system-prompt.ts`:**
- Purpose: AI personality configuration
- Generated: No (hand-written)
- Committed: No in production (.gitignored)—use env var instead
- Local development: Required for full functionality

**`.vercel/`:**
- Purpose: Deployment metadata
- Generated: Yes (by Vercel on deploy)
- Committed: Partially (project.json tracked, cache/ not)
- Note: Do not manually edit

**`.planning/codebase/`:**
- Purpose: GSD project documentation
- Generated: Yes (by `/gsd-map-codebase` command)
- Committed: No (.gitignore)
- Contents: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, CONCERNS.md, STACK.md, INTEGRATIONS.md

## Build & Deployment

**Local Development:**
```bash
npm run dev:local    # Concurrently: tsup --watch + http-server on :8080
npm run lint-and-format  # Biome check/fix
```

**Production Build:**
```bash
npm run build        # tsup --minify → dist/index.global.js
# Vercel auto-deploys on push to main
```

**Environment Setup:**
- `.env` file required locally with:
  - `ANTHROPIC_API_KEY=sk-...` (for local API testing via Vercel serverless)
  - `SYSTEM_PROMPT=...` (Ignacio's PROMETHEUS personality—kept in .gitignore)
- Vercel environment variables set via dashboard (production deployment)
